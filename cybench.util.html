cybench.util.optuna_hyper_opt module

class cybench.util.optuna_hyper_opt.OptunaOptimizer(cfg, dataset: BaseDataset, path: str | Path, study_name: str, split_dataset_years: set[Any] | list[Any] | None = None)

Bases: object

Hyperparameter Optimizer using Optuna.

This class handles the creation of an Optuna study, the definition of the objective function based on a Hydra configuration with recursive _search_ keys, and the storage of the optimal configuration.

Parameters:

  • cfg – Configuration containing hp_config (Optuna settings), val_cfg (for val-splitting) & model_cfg (model plus _search_ keys to be optimized)

  • dataset (BaseDataset) – The dataset instance to be used for optimization.

  • path (str) – Directory path where the optimal_model.yaml will be stored.

  • study_name (str) – Unique identifier for the Optuna study.

_assign_gpu() → str

Pick the GPU with the fewest active trials.

_objective(trial: Trial) → float

The objective function for Optuna. 1. Clones the base config. 2. Samples parameters using the trial. 3. Instantiates the model. 4. Performs validation (CV or Hold-out). 5. Returns the mean validation metric.

_release_gpu(device: str) → None

Decrement counter when trial finishes or fails.

_resolve_search_space(cfg: DictConfig | Any, trial: Trial | FixedTrial, prefix: str = '') → None

Recursively traverses the config, finds _search_ keys, samples values from Optuna, updates the config in-place, and removes the _search_ key.

optimize() → DictConfig

Runs the Optuna optimization process.

Returns:

The best model configuration found, with _search_ keys

resolved to their optimal values.
